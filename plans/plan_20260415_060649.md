# Plan: Load Real Orders from Backend

**Created:** 2026-04-15 06:06:49
**Status:** Approved
**Complexity:** Medium

---

## Description

Replace the hardcoded mock orders in the Orders page with real data from the backend API. This involves creating the Order model and API routes in the backend (similar to the existing Products implementation), then connecting the frontend Orders page to consume this API.

**User Preferences:**
- Use auto-increment for order IDs (no custom order_number field)
- Keep it simple (no user association for now)
- Keep it simple (no order items details, just item_count)
- Seed 5 sample orders

---

## Steps

### Backend Implementation

#### Step 1: Create Order Model
**File:** `backend/models/order.py`

Create `Order` SQLAlchemy model with fields:
- `id` (Integer, primary key, auto-increment)
- `customer_name` (String(100))
- `customer_email` (String(100))
- `total_amount` (Float)
- `status` (String(20), default: "pending")
- `item_count` (Integer, default: 1)
- `created_at` (DateTime, default=datetime.utcnow)
- `updated_at` (DateTime, default=datetime.utcnow, onupdate=datetime.utcnow)

Status enum values: pending, processing, shipped, delivered, cancelled

#### Step 2: Create Orders API Routes
**File:** `backend/routes/orders.py`

Create FastAPI router with prefix `/api/orders`

Implement endpoints:
- `GET /api/orders` - List all orders with optional status filtering (?status=shipped)
- `GET /api/orders/{order_id}` - Get specific order by ID
- `POST /api/orders` - Create new order
- `PUT /api/orders/{order_id}` - Update order (status, customer info)
- `DELETE /api/orders/{order_id}` - Delete order (soft delete by setting status to "cancelled")

Use Pydantic schemas for request/response validation:
- `OrderBase` - customer_name, customer_email, total_amount, item_count
- `OrderCreate` - inherits OrderBase + status (optional)
- `OrderUpdate` - inherits OrderBase (all optional)
- `OrderResponse` - all fields including id, created_at, updated_at

#### Step 3: Register Orders Router
**File:** `backend/main.py`

- Import orders router: `from routes.orders import router as orders_router`
- Include router: `app.include_router(orders_router, prefix="/api/orders", tags=["orders"])`
- Add to lifespan function: seed 5 sample orders on startup
  - Order 1: "Alice Johnson", alice@example.com, $125.99, processing, 3 items
  - Order 2: "Bob Smith", bob@example.com, $89.50, shipped, 2 items

  - Order 4: "David Brown", david@example.com, $45.99, pending, 1 item
