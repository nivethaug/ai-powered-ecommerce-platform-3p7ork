# PLAN: Analytics Backend Integration

**Created**: 2026-04-15 07:00:00
**Status**: APPROVED

---

## Description

Integrate the Analytics page with real data from the backend database by:

1. Creating a new Analytics API in the backend to calculate metrics from Orders, Products, and Customers
2. Creating an analytics service in the frontend to consume the API
3. Updating the Analytics page to use real data with React Query
4. Supporting date range filtering and time-based aggregations

---

## Steps

### Step 1: Create Backend Analytics Service

**File**: `backend/services/analytics_service.py` (NEW)

- Create service with methods to calculate:
  - Total revenue, orders, customers from database
  - Average order value
  - Growth rates (compare current period vs previous period)
  - Revenue/orders grouped by day, week, month
  - Revenue grouped by product category
- Use SQLAlchemy queries with date filtering
- Support different time periods (7d, 30d, 90d, 1y)

### Step 2: Create Backend Analytics Routes

**File**: `backend/routes/analytics.py` (NEW)

- Create endpoints:
  - `GET /api/analytics/overview` - Key metrics (revenue, orders, customers, AOV, growth rates)
  - `GET /api/analytics/revenue-by-period` - Revenue data grouped by time period
  - `GET /api/analytics/orders-by-period` - Orders data grouped by time period
  - `GET /api/analytics/revenue-by-category` - Revenue by product category
- Query params: `period` (7d, 30d, 90d, 1y), `start_date`, `end_date`

### Step 3: Register Analytics Router

**File**: `backend/main.py`

- Import and include analytics router
- Add to app routers

### Step 4: Update API Configuration

**File**: `frontend/src/lib/api-config.ts`

- Add analytics endpoints:
  ```typescript
  analytics: {
    overview: "/api/analytics/overview",
    revenueByPeriod: "/api/analytics/revenue-by-period",
    ordersByPeriod: "/api/analytics/orders-by-period",
    revenueByCategory: "/api/analytics/revenue-by-category",
  }
  ```

### Step 5: Create Frontend Analytics Service

**File**: `frontend/src/services/database.ts`

- Add `analyticsService` with methods:
  - `getOverview(params)` - Fetch key metrics
  - `getRevenueByPeriod(params)` - Fetch revenue chart data
  - `getOrdersByPeriod(params)` - Fetch orders chart data
  - `getRevenueByCategory(params)` - Fetch category data

### Step 6: Update Analytics Page

**File**: `frontend/src/pages/Analytics.tsx`

- Replace hardcoded mock data with React Query hooks
- Use `analyticsService` methods to fetch real data
- Pass date range as query parameter
- Calculate growth rates from backend data
- Handle loading/error states

## Design Decisions

1. **Date Range Filtering**: Server-side with query params for better performance
2. **Growth Rate Calculation**: Current period vs preceding period (e.g., last 7 days vs previous 7 days)
3. **Category Revenue Logic**: Simplified approach - group orders by product category and sum total_amount (assumes single category per order for now)
4. **Authentication**: Require authentication (consistent with other endpoints)
